Chapter 1
What is RAG?
The big idea explained in plain English before any code

🤔 The Problem RAG Solves

Imagine you upload a 200-page company annual report and ask: "What is the company's AI strategy?" A regular LLM (like ChatGPT) has two problems:

  • It can't read your file — it only knows what it was trained on (data from the internet, cut off at a certain date)
  • Even if it could, 200 pages is too long to fit in a single prompt (LLMs have a "context window" limit)

RAG (Retrieval-Augmented Generation) solves this by splitting the task into two parts:

  • Retrieval — Find only the relevant parts of the document (not the whole thing)
  • Generation — Feed those parts to the LLM and let it write a grounded answer
💡
The Perfect Analogy
Think of RAG like an open-book exam. Without RAG = answering from memory (LLM might hallucinate). With RAG = you can look up specific pages in your textbook before answering. The LLM reads those specific pages and writes a precise, sourced answer.

pdf_processor.py
Step 1 of RAG: Read the document and split it into smart chunks

🎯 What This File Does

Before any AI magic can happen, we need to read the PDF and break it into small pieces. This file has two functions that do exactly that.

Function 1: extract_text_from_pdf()

import fitz  # PyMuPDF — fitz is the internal name for pymupdf

def extract_text_from_pdf(pdf_path: str) -> list:
    """Open a PDF and extract text from every page."""
    
    doc = fitz.open(pdf_path)  # Load PDF into memory
    # doc is now a PDF document object
    # len(doc) = number of pages
    
    pages = []
    for page_num in range(len(doc)):    # Loop: 0, 1, 2, ... n-1
        page = doc[page_num]               # Get page object by index
        text = page.get_text()           # Extract all text from that page
        
        if text.strip():                  # Skip empty pages
            pages.append({
                "page_num": page_num + 1,  # +1: pages start at 1, not 0
                "text": text.strip()         # .strip() removes extra whitespace
            })
    
    doc.close()  # Always close the file when done
    return pages
ℹ️
Why fitz instead of pymupdf?
The package is installed as pymupdf but imported as fitz. This is historical — the library was originally called "fitz" (named after the PDF spec author). Both names work: import fitz and import pymupdf import the same library.

Function 2: chunk_text()

Now here's the clever part. We can't send 200 pages to the LLM — it's too long. We split the text into overlapping chunks.

def chunk_text(pages, chunk_size=200, overlap=30):
    """Split pages into overlapping word chunks."""
    chunks = []
    chunk_id = 0
    
    for page_data in pages:             # Loop through each page
        words = page_data["text"].split() # Split text into individual words
        # e.g., "Hello world" → ["Hello", "world"]
        
        i = 0
        while i < len(words):
            chunk_words = words[i : i + chunk_size]  # Grab 200 words
            chunk_text = " ".join(chunk_words)        # Rejoin as string
            
            chunks.append({
                "chunk_id": chunk_id,
                "page_num": page_data["page_num"],
                "text":     chunk_text,
                "word_start": i
            })
            chunk_id += 1
            
            i += chunk_size - overlap  # Move forward by 170 (200-30)
            # This creates overlap: last 30 words of chunk N
            # become the first 30 words of chunk N+1
    
    return chunks
💡
Why Overlap Matters
Without overlap, a key sentence split across two chunks would lose context. With 30-word overlap, the end of chunk 1 is repeated at the start of chunk 2, preserving meaning across boundaries. Imagine if "The company's Q3 revenue was $2.1 billion, representing a..." was cut in half — you'd lose the conclusion without overlap.

vector_store.py
Step 2 of RAG: Convert text to numbers and enable lightning-fast semantic search

🔢 What Are Embeddings?

An embedding is a list of numbers (a vector) that represents the meaning of a piece of text. The sentence-transformers model converts any sentence into a list of 384 numbers. 

# "Revenue grew 20% in Q3"  →  [0.23, -0.54, 0.12, ..., 0.87]  (384 numbers)
# "Sales increased by a fifth" → [0.25, -0.51, 0.14, ..., 0.85]  (very similar!)
# "The cat sat on a mat"     →  [-0.45, 0.23, -0.78, ..., 0.12] (very different!)

FAISS measures the distance between vectors. Closer vectors = similar meaning. So when you ask "What is the revenue?", FAISS finds the chunks whose vectors are closest to the question's vector.

🗄️ The VectorStore Class — Line by Line

import faiss
import numpy as np
from sentence_transformers import SentenceTransformer

class VectorStore:
    def __init__(self):
        # Load the embedding model (~80MB, downloads on first run)
        self.model = SentenceTransformer("all-MiniLM-L6-v2")
        
        self.index = None   # FAISS index — None until first chunk added
        self.chunks = []    # Stores chunk metadata (text, page_num, etc.)
        self.dimension = 384  # all-MiniLM-L6-v2 output size
    
    def add_chunks(self, chunks):
        # Step 1: Extract just the text from each chunk dict
        texts = [chunk["text"] for chunk in chunks]
        # List comprehension — Python shorthand for a for loop
        # Same as: texts = []; for c in chunks: texts.append(c["text"])
        
        # Step 2: Convert all texts to 384-dim vectors in one batch
        embeddings = self.model.encode(texts, show_progress_bar=True)
        # embeddings.shape = (num_chunks, 384)
        
        # Step 3: Convert to float32 (FAISS requirement)
        embeddings = np.array(embeddings, dtype=np.float32)
        
        # Step 4: Create FAISS index on first call
        if self.index is None:
            # IndexFlatL2 = exact search using L2 (Euclidean) distance
            self.index = faiss.IndexFlatL2(self.dimension)
        
        # Step 5: Add vectors to FAISS (assigns IDs 0, 1, 2, ...)
        self.index.add(embeddings)
        
        # Step 6: Store metadata (parallel to FAISS IDs)
        self.chunks.extend(chunks)
    
    def search(self, query, top_k=5):
        # Step 1: Embed the query (same model as chunks)
        query_embedding = self.model.encode([query])
        query_embedding = np.array(query_embedding, dtype=np.float32)
        
        # Step 2: FAISS finds top_k nearest vectors
        distances, indices = self.index.search(query_embedding, top_k)
        # distances[0] = [0.23, 0.45, 0.67, ...] (L2 distances)
        # indices[0]   = [42, 7, 113, ...]        (chunk IDs)
        
        # Step 3: Look up chunk metadata by FAISS ID
        results = []
        for i, idx in enumerate(indices[0]):
            if idx != -1:  # -1 means FAISS found fewer than top_k
                chunk = self.chunks[idx].copy()
                chunk["score"] = float(distances[0][i])
                results.append(chunk)
        
        return results  # Sorted best-first by FAISS
🎓
Interview-Ready Explanation
"How does semantic search work?"
We use sentence-transformers to map text into a 384-dimensional vector space where semantic similarity corresponds to geometric proximity. We index these vectors in FAISS using L2 distance. At query time, the question is embedded with the same model and FAISS returns the k nearest neighbors in O(n) time — no keyword matching, pure meaning-based retrieval.

groq_client.py
Step 3 of RAG: Prompt engineering + calling the LLM to generate grounded answers

🤖 What is Groq?

Groq is a company that built custom AI chips called LPUs (Language Processing Units). They run Meta's LLaMA 3 70B model — one of the most powerful open-source LLMs — at speeds of 700+ tokens/second (vs. 30-50 tokens/second on regular GPUs). And it's completely free on their free tier.

✍️ Prompt Engineering — The Most Important Skill

Prompt engineering is the art of crafting instructions that get the LLM to behave exactly as you want. In RAG, the key instruction is: "Only answer from the provided context." This prevents the LLM from "hallucinating" (making up facts). 

def build_rag_prompt(question, context_chunks):
    # Build the context section (the retrieved chunks)
    context_parts = []
    for i, chunk in enumerate(context_chunks, 1):
        context_parts.append(
            f"[Source {i} | Page {chunk['page_num']}]\n{chunk['text']}"
        )
    context = "\n\n".join(context_parts)  # Join with blank lines
    
    # The RAG prompt — three parts: instructions + context + question
    prompt = f"""You are an expert AI analyst analyzing business documents.

INSTRUCTIONS:
- Answer the question ONLY using the provided context below.
- If the context doesn't contain the answer, say so explicitly.
- Always cite which page the information comes from.

CONTEXT FROM DOCUMENTS:
{context}

USER QUESTION: {question}

YOUR ANSWER:"""
    
    return prompt
def ask_groq(question, context_chunks, model="llama3-70b-8192"):
    # Read API key from environment (.env file)
    api_key = os.getenv("GROQ_API_KEY")
    
    # Create the Groq client (like a phone connection to their servers)
    client = Groq(api_key=api_key)
    
    # Call the API — same interface as OpenAI
    response = client.chat.completions.create(
        model=model,
        messages=[
            {
                "role": "system",      # Background instructions for the AI
                "content": "You are a precise document analyst..."
            },
            {
                "role": "user",        # The actual prompt from the user
                "content": build_rag_prompt(question, context_chunks)
            }
        ],
        temperature=0.1,   # Low randomness → factual answers
        max_tokens=1024,  # Max response length (~750 words)
    )
    
    # Extract the answer text from the response object
    answer = response.choices[0].message.content
    
    return {
        "answer":      answer,
        "sources":     sorted(set(c["page_num"] for c in context_chunks)),
        "tokens_used": response.usage.total_tokens
    }

🚀 What is a REST API?

A REST API is a set of URLs (endpoints) that your app exposes. Other programs (like our Streamlit UI) can send HTTP requests to these URLs to do things like upload files or ask questions.

MethodEndpointWhat it does
GET/Health check — "Is the server alive?"
POST/uploadUpload a PDF and index it in FAISS
POST/askAsk a question, get AI answer + sources
GET/documentsList all indexed documents
GET/statsHow many chunks are indexed?
DELETE/clearRemove all documents from the index
 
from fastapi import FastAPI, UploadFile, File, HTTPException
from pydantic import BaseModel

app = FastAPI(title="DocuMind RAG API")

@app.post("/upload")
async def upload_document(file: UploadFile = File(...)):

Why async def?
async def makes the function asynchronous — it can handle multiple requests simultaneously without blocking. While one request waits for the Groq API to respond (network I/O), FastAPI can serve other requests. This makes the server much more efficient under load.

🎨 How Streamlit Works

Streamlit is magical: every Python function call becomes a UI element. But there's one key thing to understand:

⚠️
Streamlit Re-runs the Entire Script on Every Interaction
Every time a user clicks a button, types something, or uses a slider, Streamlit re-executes every line of streamlit_app.py from top to bottom. This is why we use st.session_state — it's the only way to preserve data between re-runs (like a phone's RAM that survives between app refreshes). 
import streamlit as st
import requests

# ── Must be FIRST Streamlit command ──────────────────────────────
st.set_page_config(
    page_title="DocuMind",
    page_icon="🧠",
    layout="wide"         # Use full browser width
)

# ── Session State — persists between re-runs ──────────────────────
if "chat_history" not in st.session_state:
    st.session_state.chat_history = []  # List of message dicts

# ── Sidebar — file upload + document list ─────────────────────────
with st.sidebar:
    uploaded_file = st.file_uploader("Choose a PDF", type=["pdf"])
    
    if st.button("🚀 Process Document"):
        # Call FastAPI /upload endpoint
        files = {"file": (uploaded_file.name, uploaded_file.getvalue())}
        r = requests.post("http://localhost:8000/upload", files=files)
        st.success("Indexed!")

# ── Chat interface — display history ─────────────────────────────
for msg in st.session_state.chat_history:
    if msg["role"] == "user":
        st.markdown(f"**You:** {msg['content']}")
    else:
        st.markdown(f"**AI:** {msg['content']}")

# ── Chat input — sticky bar at bottom ────────────────────────────
question = st.chat_input("Ask a question...")

if question:
    # Add to history
    st.session_state.chat_history.append({"role": "user", "content": question})
    
    # Call FastAPI /ask endpoint
    r = requests.post("http://localhost:8000/ask",
                       json={"question": question, "top_k": 5})
    result = r.json()
    
    # Save AI response to history
    st.session_state.chat_history.append({
        "role": "assistant",
        "content": result["answer"]
    })
    
    st.rerun()  # Force re-render to show new messages

How to talk about this project confidently at Celonis — questions and perfect answers

💬 Q&A: Explain Every Part at Any Level

"What is RAG and why did you build it?"

🎓
Perfect Answer
"RAG stands for Retrieval-Augmented Generation. The core problem is that LLMs have a limited context window and can't read enterprise documents at scale. RAG solves this with two phases: first, we embed document chunks into a vector space and index them in FAISS. At query time, we embed the user's question, find the semantically nearest chunks, and inject them as context into the LLM prompt. This grounds the answer in real document content and eliminates hallucination — the LLM can only cite what's in front of it."

"How does the vector similarity search work?"

🎓
Perfect Answer
"We use all-MiniLM-L6-v2, a sentence-transformer model that maps text to 384-dimensional vectors. Semantically similar text produces geometrically close vectors. FAISS stores these vectors and uses L2 (Euclidean) distance for exact nearest-neighbor search. At query time, the question is embedded with the same model and FAISS returns the k nearest chunk vectors. This is semantic search — unlike keyword matching (TF-IDF, BM25), it understands paraphrase and synonymy."

"How does this relate to Celonis's AI strategy?"

🎓
Perfect Answer
"Celonis sits on massive amounts of unstructured process data — audit trails, case notes, email chains. RAG is exactly the technology that unlocks AI value from this data without requiring it to be perfectly structured. This system demonstrates three key skills from the job description: NLP pipeline engineering, GenAI integration with production-grade APIs, and building AI interfaces that business users can interact with — no data science background required."

"Why FastAPI instead of Flask?"

ℹ️
Answer
FastAPI is async-native (built on ASGI/Starlette) vs Flask's synchronous WSGI. It provides automatic OpenAPI documentation, Pydantic validation, and dependency injection out of the box. For an AI API that makes external network calls (to Groq), async is critical — it doesn't block the event loop while waiting for LLM responses.

"What would you do to scale this to production?"

ℹ️
Answer
1. Persistent storage: Replace in-memory FAISS with Pinecone or Weaviate for multi-server durability
2. Authentication: Add JWT tokens to the FastAPI endpoints
3. Async processing: Move PDF indexing to a Celery/Redis background queue (indexing large PDFs shouldn't block the API)
4. Caching: Cache common question embeddings with Redis to avoid redundant LLM calls
5. Observability: Add LangSmith tracing to monitor RAG chain quality

🔴 Common Errors & Fixes

Error: "GROQ_API_KEY not found"

⚠️
Fix
Open dox\.env and make sure it contains exactly:
GROQ_API_KEY=gsk_your_key_here
No quotes, no spaces around the = sign. Then restart FastAPI.

Error: "🔴 API Offline" in Streamlit

⚠️
Fix
FastAPI is not running. Start it first:
.venv\Scripts\activate
uvicorn app.main:app --reload
Then refresh Streamlit.

Error: "ModuleNotFoundError: No module named 'fitz'"

⚠️
Fix
The virtual environment isn't activated. Run:
.venv\Scripts\activate
Then try again. You should see (.venv) in your terminal prompt.

Error: "sentence-transformers downloading model every time"

ℹ️
This is normal on first run
The all-MiniLM-L6-v2 model (~80MB) downloads once and is cached. After the first run, it loads from cache in <2 seconds. The cache is stored in ~/.cache/torch/sentence_transformers/.

Error: "No documents have been uploaded yet"

⚠️
Fix
The FAISS index is in-memory — it resets every time you restart FastAPI. Always upload your PDF again after restarting the server.
